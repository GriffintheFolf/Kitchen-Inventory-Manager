How to Use the Kitchen Inventory Manager

This program was designed to be easy for almost anyone to utilize; nevertheless, a brief tutorial on how the program works is provided here to clarify some of its terminology.

Adding an Item

The product name is the name of the food item you want to add. It can be either a specific name, with the brand, or a generic name; it does not matter which.

The item count is the amount of "containers" you have of an item. For example, with three boxes of crackers, you will have an item count of 3. The item count allows for decimal numbers, for example, if you have only half of one box of crackers, the item count is 0.5.

The unit amount is the amount of the item per container. Most often, this is the weight measured in grams, but can also be the volume in millilitres for liquids. In theory, any metric unit should work here. For example, if one bottle of olive oil contains 1000 mL, the unit amount is 1000, specified without units after the number.

The expiration date should be self-explanatory.

The barcode number should be the same as the barcode number on the item's packaging.

All of these values must be specified to add the item into the database.

Viewing Items

On the main page of the program, below the form to add an item, there is a table consisting of all items presently within the database.

The tabel displays the product name, item count, unit amount, expiration date, and barcode number. The table is sorted by the product name in "ASCIIbetical" order; capital letters come before lowercase letters.

Editing Items

Located next to the barcode number of the item in the main table is a link stating "Edit". Clicking on it will allow you to edit the name, item count, unit amount, and expiration date of the item.

The barcode number cannot be modified, because it is how each item is uniquely represented within the database.

Deleting Items

Clicking the link labelled "Delete" next to an item in the main table will immediately delete that item from the database. There is no confirmation prompt (yet)!
